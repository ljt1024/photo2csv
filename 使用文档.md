# 产品图片识别入库工具使用文档

本文档说明如何在当前工程目录运行 `photo2csv.py`，把商品图片识别结果写入 CSV，并保存每组商品的第一张图片。

## 1. 功能说明

脚本会完成以下工作：

- 默认读取三张一组的商品图片，也可以用 `--group-size` 改成每组两张等
- 调用千问视觉模型识别商品信息
- 运行时输入平台标识，例如 `pdd`、`ks`、`dy`、`jd`、`xhs`
- 自动生成 `平台标识 + 4位数字` 这类图片编号，例如 `pdd0001`、`ks0001`
- 将识别结果追加到当前工程目录下的 `product_data_20260525.csv`
- 将每组第一张图片保存为当前工程目录下的 `images/平台标识NNNN.jpg` 或 `.png`，格式跟随输入首图
- CSV 使用 UTF-8 带 BOM 编码，兼容 Excel 的「CSV UTF-8」格式，避免中文乱码
- 写入 CSV 前自动生成备份文件

默认输出位置：

```text
photo2csv/
├── product_data_20260525.csv
├── product_data_20260525.csv.bak.YYYYMMDD_HHMMSS
└── images/
    └── ks0001.jpg
```

## 2. 安装依赖

在当前工程目录执行：

```bash
pip install -r requirements.txt
```

首图会按输入格式保存：JPG/JPEG 输出 `.jpg`，PNG 输出 `.png`，不会强制转成 PNG。

## 3. 千问 Key 存储方式

### 推荐方式：放在 `.env`

项目根目录下使用 `.env` 保存千问 API Key：

```text
DASHSCOPE_API_KEY=sk-替换成你的真实Key
```

脚本启动时会自动读取当前工程目录下的 `.env` 文件。

### 安全要求

请遵守以下规则：

- 不要把真实 key 写进 `photo2csv.py`
- 不要把真实 key 写进 `README.md` 或其他文档
- 不要把真实 key 发给别人或贴到聊天、工单、截图里
- 不要强制提交 `.env`
- 如果怀疑 key 已泄露，立即到阿里云百炼控制台禁用或重新生成 key

当前项目的 `.gitignore` 已包含：

```text
.env
```

这意味着正常情况下 `.env` 不会被 git 提交。仍然要避免执行类似下面这种命令：

```bash
git add -f .env
```

### 可选：限制本机文件权限

如果想让 `.env` 只能由当前用户读取，可以执行：

```bash
chmod 600 .env
```

### 备选方式：临时环境变量

也可以在终端临时设置：

```bash
export DASHSCOPE_API_KEY="sk-替换成你的真实Key"
```

这种方式只对当前终端会话生效。更推荐使用 `.env`，因为不容易忘记，也不用每次重新 export。

## 4. 素材格式

### 支持品类

当前支持以下品类：

- 美妆护肤
- 食品饮料
- 个护家清
- 母婴用品
- 保健滋补
- 数码家电
- 家居百货
- 服装鞋包
- 宠物用品

### Zip 素材包

支持直接读取 `.zip` 素材包，例如：

```text
test.zip
└── test/
    ├── 1-1.jpg
    ├── 1-2.jpg
    └── 1-3.jpg
```

命名规则：

- `1-1.jpg`：第 1 组第 1 张
- `1-2.jpg`：第 1 组第 2 张
- `1-3.jpg`：第 1 组第 3 张
- 多组继续使用 `2-1.jpg`、`2-2.jpg`、`2-3.jpg`

脚本会自动忽略 macOS 压缩包里的 `__MACOSX` 和 `._*` 元数据文件。

### 图片目录

也支持读取目录：

```bash
python3 photo2csv.py --input-dir ./待处理图片
```

默认会自动识别 `1-1/1-2/1-3` 这种命名。如果没有这种命名，则按文件名排序后每组 3 张分组。

如果每组只有 2 张图片，命名规则就是 `1-1.jpg、1-2.jpg、2-1.jpg、2-2.jpg`，运行识别时加 `--group-size 2`。

### 素材重命名打包

`rename_images_zip.py` 默认按每组 3 张生成 ZIP 内文件名：

```bash
python3 rename_images_zip.py ./待处理图片 --output renamed_images.zip
```

如果每组是 2 张，添加 `--group-size 2`：

```bash
python3 rename_images_zip.py ./待处理图片 --group-size 2 --output renamed_images.zip
python3 photo2csv.py renamed_images.zip --group-size 2 --platform ks
```

## 5. 运行方式

### 预览模式

先用 `--dry-run` 检查识别结果和输出路径：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip --dry-run
```

注意：`--dry-run` 只预览，不会生成 CSV，不会创建图片文件，也不会修改任何数据。

### 正式写入

确认预览没问题后，去掉 `--dry-run`：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip
```

脚本会先提示输入平台标识：

```text
请输入平台标识（例如 pdd/ks/dy/jd/xhs）：
```

输入的平台会同时用于：

- CSV 的 `platform` 字段
- CSV 的 `image_id` 前缀
- 首图文件名，例如 `ks0001.jpg`、`dy0001.png`

如果不想交互输入，也可以直接传参：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip --platform ks
```

运行后会在当前工程目录生成或更新：

```text
product_data_20260525.csv
images/
```

如果某一组识别失败，脚本不会中断。对应图片仍会按顺序保存，CSV 也会插入同一顺序的一行错误数据：

- `title` 填 `识别失败`
- 其他识别字段尽量留空
- `remark` 写入具体错误提示，例如 `识别错误：识别结果不是合法 JSON...`

这样可以保证图片编号、CSV 行和原始素材组顺序一致。

如果当前工程目录没有 `product_data_20260525.csv`，脚本会先从下面这个模板复制一份：

```text
/Users/ljt/Downloads/product_data_20260525.csv
```

然后再追加本次识别结果。

## 6. 常用参数

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip \
  --timeout 180 \
  --retries 5
```

常用参数说明：

| 参数 | 说明 |
|------|------|
| `--dry-run` | 只预览，不写文件 |
| `--output-dir ./result` | 指定输出目录 |
| `--csv ./data.csv` | 指定目标 CSV |
| `--images-dir ./images` | 指定图片输出目录 |
| `--template-csv PATH` | 指定初始化用的模板 CSV |
| `--group-size 2` | 指定每组图片数量；默认 3 |
| `--group-mode name` | 指定分组方式；默认 auto |
| `--platform ks` | 指定平台标识；不传时脚本会提示输入 |
| `--model qwen3-vl-plus` | 指定千问视觉模型 |
| `--timeout 180` | 增加请求超时时间 |
| `--retries 5` | 增加失败重试次数 |
| `--no-verify-ssl` | 临时跳过证书校验，用于排查 SSL 问题 |
| `--copy-first-image` | 复制第一张图，而不是移动 |
| `--no-backup` | 写 CSV 前不生成备份 |

## 7. SSL 或网络错误处理

如果遇到类似错误：

```text
调用千问识别失败：EOF occurred in violation of protocol
```

先重试：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip --timeout 180 --retries 5
```

如果仍然失败，可以临时排查证书或代理问题：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip --timeout 180 --retries 5 --no-verify-ssl
```

`--no-verify-ssl` 只建议排障时使用。长期使用应优先检查本机网络、代理、VPN 或证书环境。

## 8. 没有 Key 时的手动模式

如果暂时不调用千问，也可以把识别结果写到 JSON 文件，然后导入：

```json
[
  {
    "category": "食品饮料",
    "has_sku_info": 1,
    "title": "某品牌大米5斤装",
    "brand": "某品牌",
    "sku_spec": "5斤",
    "price": 29.9,
    "price_type": "闪降价",
    "shop_name": "某品牌旗舰店",
    "remark": ""
  }
]
```

执行：

```bash
python3 photo2csv.py /Users/ljt/Downloads/test.zip --manual-json result.json
```

## 10. 快速排查清单

如果当前目录没有生成文件，先检查：

- 是否用了 `--dry-run`
- 千问识别是否报错并提前退出
- 终端当前路径是否是 `/Users/ljt/Desktop/my-app/photo2csv`
- 输出日志里的 `输出 CSV` 和 `输出图片目录` 是否指向当前工程目录
- `.env` 是否存在，且包含 `DASHSCOPE_API_KEY=...`
- `product_data_20260525.csv` 是否已有权限问题或被其他软件占用
